BrainX 노트 페이지 수정 작업 가이드
작업 시작 전 필수 규칙

현재 노트 페이지는 BrainX의 핵심 기능 페이지이며,

다음 기능들은 이미 구현되어 있거나 여러 차례 버그 수정 및 UX 개선이 진행된 상태이다.

따라서 새로운 기능 구현보다

기존 기능의 회귀(regression) 방지를 최우선 목표로 한다.

절대 금지

다음 행위는 금지한다.

1. 추정 수정 금지

코드를 열어보지 않고

"아마 이럴 것이다"

"이 구조일 것이다"

라고 가정하여 수정 금지.

반드시 실제 구현 확인 후 수정.

2. 대규모 리팩토링 금지

작업 범위와 무관한:

컴포넌트 분리
폴더 구조 변경
상태관리 변경
에디터 구조 변경

금지.

필요하다면 사유를 먼저 보고.

3. 동작하는 기능 삭제 금지

현재 동작하는 기능을

"안 쓰는 것 같다"

"불필요해 보인다"

라는 이유로 제거 금지.

실제 사용 여부 확인 후 보고.
